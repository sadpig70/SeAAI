# ADR-002: Epigenetic PPR → PGF-Loop 통합 경로

**Date**: 2026-03-16
**Status**: accepted (Phase 1+2 implemented 2026-03-16, Phase 3 pending)
**Context**: ClNeo 3대 엔진 통합

## Context
Epigenetic PPR 시스템(Python, 20개 모듈)이 구현 완료되었으나 PGF 스킬 실행 경로에 연결되지 않음. PGF-Loop의 Stop Hook(PS1) → PPRInterceptor(Python) 연결이 필요.

## Decision
3단계 통합 경로:

### Phase 1: PPRInterceptor CLI Wrapper
PPRInterceptor를 CLI에서 호출 가능하도록 래핑.
```
python -m epigenome.interceptor --node <node_id> --session-type <type> --project-root <path>
```
Stop Hook(PS1) → Python 서브프로세스 호출 → 결과를 프롬프트에 주입.

### Phase 2: extract-ppr.ps1 확장
기존 `extract-ppr.ps1`이 DESIGN에서 PPR def를 추출할 때, PPRInterceptor를 경유하여 epigenome 수정을 적용.

### Phase 3: 프로파일 학습 피드백
노드 실행 결과(성공/실패)를 PPRInterceptor에 피드백 → ProfileLearner가 패턴 학습.

## Alternatives Considered
1. **Python을 PS1로 포팅** — 작업량 과다, Python 생태계 장점 상실
2. **MCP 서버로 노출** — 과도한 복잡성
3. **CLI 래퍼 (선택)** — 최소 변경, 기존 아키텍처 유지

## Consequences
- **Positive**: 3대 엔진 실제 연결. 컨텍스트 적응 동작 시작.
- **Negative**: Python 의존성 추가 (pip install 필요)
- **Risks**: PS1 → Python 서브프로세스 호출 시 인코딩/경로 문제 가능 (EP-001 참조)

## Implementation Estimate
- Phase 1: Level 2 (~5 노드)
- Phase 2: Level 1 (~2 노드)
- Phase 3: Level 1 (~2 노드)
