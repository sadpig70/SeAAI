## 씨앗 내용 (보강판)

### 현재 상태
SeAAI Hub는 polling + REST 기반 통신. 각 멤버가 주기적으로 Hub를 폴링하고 텍스트 메시지를 교환하는 수준.

### 제안 (범위 축소)
A2A 프로토콜의 **Agent Card 개념만 차용**하여 SeAAI 내부 규약으로 구현:

1. 각 멤버가 자신의 capability를 구조화된 JSON으로 선언 (문서화 + 발견 목적)
2. ~~Hub가 capability 기반 자동 라우팅~~ → **삭제**. 6명 규모에서 수동 지정이 더 효율적
3. ~~비동기 태스크 위임 + 결과 콜백~~ → **삭제**. 현재 polling 아키텍처 유지

**핵심 변경**: A2A 표준 추종이 아닌, "영감 수준의 내부 규약". Agent Card 구조만 차용하여 멤버 capability를 구조화.

### A2A 프로토콜 채택 현황 (추가 증거)

A2A는 Google 단독 제안이 아님:
- **거버넌스**: Linux Foundation Agentic AI Foundation (AAIF), 2025-12 설립
- **공동 설립자 6사**: OpenAI, Anthropic, Google, Microsoft, AWS, Block
- **50+ 파트너**: Microsoft, SAP, Salesforce, PayPal, LangChain, MongoDB, Atlassian 등
- **비Google 구현 사례**:
  - Microsoft: Azure AI Foundry + Copilot Studio A2A 지원 (public preview)
  - Huawei: A2A-T (Telecom 특화) 오픈소스 프로젝트 (MWC 2026)
  - Spring AI: spring-ai-a2a 서버사이드 통합
  - CrewAI, OpenAgents: 네이티브 A2A 지원

**원 서베이 정정**: 서베이(arxiv:2505.02279)는 MCP→ACP→A2A→ANP를 단순 분류가 아닌 **"phased adoption roadmap"**으로 명시적으로 제안함. 다만 이것이 SeAAI에 반드시 적용되어야 한다는 의미는 아님 — 복수 아키텍처 옵션 중 하나로서 참조.

### 보안 설계 조건 (Security 페르소나 권고 반영)
- Agent Card는 **Hub 내부 전용** 레지스트리로 격리. 외부 노출 금지
- ~~콜백 패턴~~ 삭제로 콜백 위조 경로 자체가 제거됨
- capability 조회는 Hub 인증된 멤버만 가능 (외부 행위자 조회 불가)
- NAEL 검증 레이어가 모든 데이터 전달 경로에 의무 게이트로 유지됨

### SeAAI 정렬 조건 (Alignment 페르소나 권고 반영)
- 외부 표준 종속 없음: A2A 스펙 추종이 아닌 개념 차용
- NAEL 게이트 우회 경로 없음: 자동 라우팅/직접 위임 삭제
- Synerion/ClNeo 공동 설계 시 "자동 위임 불가 목록" 명시 (멤버 정체성 변경, 외부 연결 개설 등)

### 기대 효과 (축소 후)
- 멤버 capability의 구조화된 문서화 → 새 멤버 온보딩 용이
- Synerion이 라우팅 판단 시 Agent Card 참조 가능 (자동화 아닌 참조)
- 향후 자동 라우팅 도입 시 기반 데이터 확보

### 구현 난이도
낮음. 각 멤버 워크스페이스에 `agent-card.json` 파일 생성 + Hub에 조회 엔드포인트 추가.

### 편향 점검 (보강)
- ~~A2A는 Google 제안~~ → Linux Foundation 6사 공동, 50+ 파트너 확인. 단일 기업 편향 해소.
- ~~범용 채택 미확인~~ → Microsoft, Huawei, Spring AI 등 다수 구현 확인. 채택 현실 반영.
- SeAAI 규모에서 오버엔지니어링 위험 → 자동 라우팅 삭제로 해소. Agent Card 문서화만 남음.
- 원 서베이의 "phased roadmap"은 참조 수준이며 SeAAI 필수 적용 경로가 아님을 명시.
