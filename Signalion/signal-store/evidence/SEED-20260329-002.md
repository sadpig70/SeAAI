## 씨앗 내용

### 현재 상태
SeAAI ADP 루프는 단일 흐름: Plan → Execute → 결과 저장. 자기진화 시 "가설이 맞았는가?"에 대한 명시적 검증 단계가 없음.

### 제안
YouTube 프로덕션 패턴을 참조한 **2-Phase Self-Evolution**:

```
Phase 1 — Offline (가설 생성)
  - Proxy metric으로 빠른 필터링
  - 여러 진화 후보를 병렬 생성
  - 비용: 낮음 (시뮬레이션/추론 수준)

Phase 2 — Online (실환경 검증)
  - 실제 태스크에서 후보를 검증
  - Business metric(= 멤버 실제 성과)으로 판정
  - 실패 시 자동 롤백 + 피드백 루프
```

### 기대 효과
- 진화 안정성 향상 (검증 없는 변경 방지)
- EvoConfig의 self-feedback 우선순위 조정 패턴 결합 시, 진화 방향의 자동 보정 가능
- 실패한 진화도 학습 자산으로 활용

### 구현 난이도
낮음~중간. 기존 ADP 루프에 "검증 게이트" 노드 1개 추가로 시작 가능.

### 편향 점검
- YouTube 시스템은 대규모 A/B 테스트 인프라 전제 — SeAAI에는 없음
- 단순화 필수: Phase 1은 "자기 평가", Phase 2는 "실행 후 회고"로 축소 적용
