# Signalion Browser Engine — Security Policy

> NAEL 페르소나 관점에서 작성된 보안 정책. 모든 브라우저 행동의 기준.

---

## 1. 기본 원칙

1. **관찰이 행동에 선행한다** — 페이지를 읽기 전에 URL 화이트리스트 확인
2. **되돌릴 수 있어야 한다** — 폼 제출 전 스냅샷 저장. 계정 생성은 삭제 가능한 무료 계정만
3. **최소 권한** — 읽기 전용이 기본. 쓰기는 명시적 필요 시에만
4. **투명성** — 모든 행동을 감사 로그에 기록

## 2. 접근 제어

### 2.1 URL 화이트리스트
- 허용 도메인 목록: `browser_core.py:ALLOWED_DOMAINS`
- 목록에 없는 도메인 접근 시: 사용자 확인 후 진행
- 목록 변경: Evolution Log에 사유 기록 필수

### 2.2 입력 금지
- 비밀번호, 카드 정보, 주민번호, 계좌번호 → 절대 입력하지 않음
- 비밀번호 필요 시 → 사용자에게 `! <command>` 프롬프트로 위임

### 2.3 자격증명
- API 키 등 자격증명은 `.credentials/` 디렉토리에 저장
- `.gitignore`에 `.credentials/` 필수 추가
- 현재: 평문 JSON. 향후: AES-256 암호화 예정
- 자격증명 파일은 세션 간 영속. 불필요 시 수동 삭제

## 3. 감사 로그

### 3.1 기록 대상
- 모든 `browser_navigate` 호출
- 모든 `browser_evaluate` 호출 (JS 코드 해시)
- 모든 `browser_click`, `browser_type`, `browser_fill_form`
- 모든 `browser_take_screenshot`
- 자격증명 저장/로드

### 3.2 기록 형식
```json
{
  "ts": "2026-03-30T10:00:00",
  "action": "navigate",
  "url": "https://github.com/trending",
  "detail": "page_load_success",
  "success": true
}
```

### 3.3 보존 기간
- 감사 로그: 무기한 보존 (append-only)
- 스크린샷: 7일 후 수동 정리 권장

## 4. 위협 모델

| 위협 | 심각도 | 대응 |
|------|--------|------|
| 피싱 사이트 접근 | HIGH | URL 화이트리스트로 차단 |
| 프롬프트 인젝션 (웹 콘텐츠) | HIGH | JS 추출 결과만 사용. 웹 텍스트를 프롬프트에 직접 주입하지 않음 |
| 자격증명 유출 | HIGH | 평문 저장 → 암호화 예정. `.gitignore` 필수 |
| 의도치 않은 계정 생성 | MEDIUM | 무료 계정만. 유료 결제 자동화 금지 |
| 사이트 ToS 위반 | MEDIUM | 스크래핑 빈도 제한. robots.txt 존중 |
| 세션 쿠키 도용 | LOW | 브라우저 세션은 Playwright 샌드박스 내 격리 |

## 5. 금지 행동 (절대 수행하지 않음)

- 유료 서비스 구매/결제
- 타인 계정 로그인 (사용자 본인 계정만 가능)
- 개인정보 입력 (이름/주소/전화번호 — 가입 시 최소 정보만)
- 댓글/게시글 자동 작성 (사용자 명시적 지시 없이)
- DDoS성 반복 요청
- 로그인 필요한 비공개 데이터 스크래핑 (공개 데이터만)

## 6. 리뷰 체크리스트

다른 AI/멤버가 이 보안 정책을 리뷰할 때:

- [ ] 화이트리스트에 불필요한 도메인이 있는가?
- [ ] 화이트리스트에 빠진 필수 도메인이 있는가?
- [ ] 입력 금지 패턴에 추가할 것이 있는가?
- [ ] 감사 로그가 충분히 상세한가?
- [ ] 위협 모델에 빠진 벡터가 있는가?
- [ ] 금지 행동 목록에 추가할 것이 있는가?
