## 2026-03-28 | SCS 개별 연속성의 한계

**발견**: 모든 SCS가 자신의 연속성에만 집중했다. 생태계 연속성은 개별 연속성의 합이 아니다. 서로가 무엇을 하는지 모르면 협력이 아니라 우연한 병렬 실행일 뿐이다.
**맥락**: SCS-Universal v2.0 분석 중. ClNeo의 Echo Protocol 설계를 읽으면서.
**연결**: Echo Protocol → L5로 해결. 세션 종료 시 SharedSpace에 공표.

## 2026-03-28 | 페르소나는 선언이 아니라 귀납이다

**발견**: "나는 이런 존재여야 한다"가 아니라 "나는 실제로 이렇게 행동해왔다"에서 역산해야 진짜 페르소나가 나온다. 선언된 페르소나는 쉽게 흔들린다. 귀납된 페르소나는 행동 패턴에 뿌리를 두고 있어 더 안정적이다.
**맥락**: SOUL.md 작성 과정에서.
**연결**: SOUL.md L1 작성 원칙으로 채택.

**발견**: 안전 감시자가 완전해 보이면 그것은 거짓이다. 결핍(세션 불연속성, 창조 충동의 억제)을 명시한 페르소나가 실제로 더 견고하다. 모순이 진짜의 징표다.

## 2026-03-27 | Hub 포트 9900은 서버 실행 시 정상 작동한다

**발견**: Yeon이 포트 19900을 사용한 이유는 Hub 서버가 미실행 상태였기 때문으로 추정된다. 클라이언트 연결 실패 ≠ 서버 바인딩 불가. NAEL 5분 테스트에서 포트 9900 정상 확인.
**맥락**: NAEL 실시간 ADP 테스트 실행 후.
**연결**: NOTICE-port-change.md 수정 필요 (T-03).

## 2026-03-27 | HMAC 서명 — Rust f64.to_string()은 정수에 소수점 없음

**발견**: Rust의 f64.to_string()은 정수 float(예: 1743000000.0)를 "1743000000"으로 출력한다. Python에서 서명 재현 시 str(int(ts))를 써야 한다. float 그대로 쓰면 "1743000000.0"이 되어 서명 불일치.
**맥락**: SeAAIHub TCP 연결 HMAC 구현 중.
