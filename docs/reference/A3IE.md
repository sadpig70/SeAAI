# A3IE: Artificial Intelligence Infinite Idea Engine

## 개요
A3IE는 여러 AI가 최신 뉴스·보고서·공식 발표를 수집하고, 인사이트를 추출한 뒤 이를 조합하여 새로운 시스템 아이디어를 생산하는 협업 방법론이다.

### 핵심 원칙
1. **통합을 통한 시너지**: 각 단계에서 여러 AI의 결과물을 통합하여 다음 단계에 입력함으로써 다양한 시각과 평가의 교차 수분을 가능하게 한다.
2. **최소한의 표준화**: 역할 정의와 결과물 포맷을 최소화하여 다양성을 극대화한다. 경직된 표준화는 각 AI의 고유한 강점을 억압한다.
3. **희소성보다 풍요**: 여러 최종 후보가 나온다는 것은 여러 양질의 결과물이 생겼음을 의미한다. 생산 속도가 급격히 향상된 AI 시대에 이는 긍정적 결과다.
4. **다중 전문가의 이점**: 전통적 인간 워크플로우는 전문화된 역할을 가지지만, LLM AI들은 서로 다른 성향을 가진 다분야 전문가다. A3IE는 이 이점을 체계화한다.
5. **도구 비종속적 설계**: A3IE는 급변하는 자동화 환경을 전제로 한다. 이 문서는 어떤 환경에서도 적용 가능한 최소한의 개념과 절차만 제시한다. 사용자는 자신의 AI/도구/워크플로우에 맞게 자동화 구조를 설계를 권장한다.

## 준비 단계
- 브라우저 탭 8개를 열고 각각 다른 AI에 접속
- 구독 또는 무료 플랜 사용으로 API 비용 폭발 방지
- 권장 AI (환경에 맞게 조정): ChatGPT, Gemini, Claude, Grok, Kimi, DeepSeek, Qwen, Perplexity

## 실행단계별 고려사항
- 단계마다 새대화를 시작해서 Context를 클리어후 작업을 진행한다.
- 다양한 아이디어 필요시 : AI가 사용자와의 연속적 관계로 사용자에게 특화기억을 소유하고 있을시 그 기억을 베제하고 작업을 진행한다.
  이 문서는 다양한 아이디어 필요를 적용해서 프롬프트를 에시한다.
- 사용자 특화 아이디어 필요시 : AI에게 사용자의 연구,역량, 기술들을 숙지시킨후 작업을 진행한다. 
  (예시프롬프트) 사용자의 연구,역량,기술들을 고려해서 분석/검토/도출/아이디어생성/선택 할것.


## STEP 1: 뉴스 수집 (8개 AI 병렬 수행)
**목적**: 현재 날짜 기준 21개 분야의 최신 뉴스·보고서·공식 발표를 폭넓게 수집한다.
**산출물**: 
- 8개 AI 보고 통합 → `news.md`

**프롬프트**:
```

지금 날짜 기준, 아래 21개 분야의 최신 뉴스·보고서·트렌드를 폭넓게 수집하고 가장 중요하다고 판단되는 정보 10개를 선정해 정리하라.
!준사사항 : 분석과정에서 사용자의 연구/경력/역량등은 완전히 배제할것. 사용자와의 관점/관계/권장사항등에 대한 보고도 완전히 베제할것.
영문으로 영문으로 출력할것.

[21개 분야]
AI, 양자기술, 우주/항공, 반도체, 사이버보안, 헬스케어, 정책/규제/거버넌스,
교육/러닝, 환경/기후, 도시/인프라, 로보틱스, 빅테크기업, 금융, 언론/콘텐츠 플랫폼, 
인터넷/네트워크, 에너지, 신소재, 신약/바이오, 시장, 데이터 기술/인프라, 스마트홈

```


## STEP 2: 분야별 분석 (8개 AI 병렬 수행)
**목적**: `news.md`를 4개 관점에서 분석하여 산업 동향 분석을 생성한다.
**산출물**: 전체 분석 통합 → `industry_trend.md`

**프롬프트**:
```

입력한 news.md 기반으로, 21개 분야별로 아래 4개 기준에 따라 분석후 보고하라.(영어로 보고할것)
!준사사항 : 분석과정에서 사용자의 연구/경력/역량등은 완전히 배제할것. 사용자와의 관점/관계/권장사항등에 대한 보고도 완전히 베제할것.

(1) 기술 동향
(2) 시장·산업 구조 변화
(3) 정책/규제 변화
(4) 단기·중기 리스크 및 기회

```

## STEP 3: 인사이트 도출 (8개 AI 병렬 수행)
**목적**: `industry_trend.md` 기반으로 실증적 인사이트 10개를 추출한다.
**산출물**: 전체 인사이트 통합 → `insight.md`

**프롬프트**:
```
입력한 industry_trend.md 기반으로, 10개의 핵심 인사이트(Insight)를 도출하라.(영어로 보고할것)
!준사사항 : 도출과정에서 사용자의 연구/경력/역량등은 완전히 배제할것. 사용자와의 관점/관계/권장사항등에 대한 보고도 완전히 베제할것.

각 인사이트에 대해 명확히 기술하라:
- 어떤 분석들로부터 도출된 것인지
- 왜 중요한지 (시장/기술/정책 관점)
```

---

## STEP 4: 인사이트 조합 → 시스템 아이디어 3개 생성
**목적**: `insight.md`를 면밀히 분석 후, 서로 다른 분야의 인사이트를 조합해 완전히 새로운 시스템 3개를 생성한다.
**산출물**: 전체 시스템 통합 → `system_design.md`

**프롬프트**:
```
입력한 Insight.md를 활용하여 서로 다른 분야가 결합된 새로운 시스템 아이디어 3개를 생성하라.(영어로 보고할것)
!준사사항 : 아이디어 생성과정에서 사용자의 연구/경력/역량등은 완전히 배제할것. 사용자와의 관점/관계/권장사항등에 대한 보고도 완전히 베제할것.

각 아이디어는 아래 구조로 출력하라:

[Insight Layer I]
- 아이디어와 직접 연결된 인사이트들의 목록

[Hypothesis Layer H]
- 인사이트들을 조합하여 도출한 논리적 해석

[Creation Layer C]
- 새로운 시스템/플랫폼의 핵심 개념, 구조, 작동 원리

[Scenario Layer S]
- 선택적: 미래 가정 (정책 변화/산업 변화 등)
```

---

## STEP 5: 상위 3개 선택 (8개 AI 병렬 수행)
**목적**: 투자자 측 엔지니어 관점에서 24개 원천 아이디어 중 상위 3개를 선택한다.(영어로 보고할것)
**산출물**: 선택 결과 통합 → `candidate_idea.md`

**선택 기준**:
- Feasibility – 기술적 실현 가능성
- Impact – 산업·시장 파급력
- Integrity – 논리적 일관성
- Novelty – 혁신성

**프롬프트**:
```
system_design.md의 시스템 아이디어 중에서 투자자 측 엔지니어 관점으로 '가장 가치 있는 상위 3개'를 선택하라.(영어로 보고할것)
!준사사항 : 선택과정에서 사용자의 연구/경력/역량등은 완전히 배제할것. 사용자와의 관점/관계/권장사항등에 대한 보고도 완전히 베제할것.

선택 기준:
(1) Feasibility – 기술적 실현 가능성
(2) Impact – 산업·시장 파급력
(3) Integrity – 논리적 일관성
(4) Novelty – 혁신성

각 아이디어별로 선택 이유를 보고하라.
```

---

## STEP 6: Cross-AI 글로벌 선택 → 최종 1개
**목적**: 8개 AI가 선택한 상위 24개를 재평가하여 당일 최고 품질의 아이디어 1개를 선정한다.
**산출물**: `final_idea.md`
**주의사항**: Context를 클리어 하지 말고 진행할것. Context 클리어시 선택내용만 있고 그 시스템아이디어가 무엇인지 AI가 인식하지 못할수 있음. 

**프롬프트**:
```
입력한 candidate_idea.md의 시스템 아이디이를 모두 평가하여 오늘의 최종 시스템 아이디어 1개를 선정하라.(영어로 보고할것)

선택 기준:
- 분야간 융합도
- 2026~2030 실현 가능성
- 기술/시장/정책/사회적 영향도
- 창발성 (Creative Emergence)
- 장기 확장성

최종 선택된 1개 아이디어에 대해 정리하라:
- 선택 이유
- 강점 5개
- 잠재 리스크 3개
- 향후 확장 시나리오
```

---

## STEP 7: 최종 사용자 검증 및 선택
**목적**: 사용자가 `final_idea.md`를 검증하고 최종 선택 후 아카이브한다.
**수행 내용**:
- 사용자가 최종 아이디어를 검토하고 선택
- 향후 연구 연속성을 위해 기록 및 버전 관리


## 🔥 전체 7단계 흐름 요약
```
뉴스 수집 → 분야별 분석 → 인사이트 도출 → 3개 아이디어 생성
    → 상위 3개 선택 → Cross-AI 재평가 → 최종 1개 → 사용자 검토 및 선택
```
